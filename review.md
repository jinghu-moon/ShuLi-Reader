## 阶段 R1 · 安全审查

> 由开发者或安全负责人代码审查完成，无需真实设备。

### 加密实现

- [ ] **R1-1** `KeyDerivation` 的 `iterations` 参数确认为 600,000（不可低于此值），检查 `KeyDerivationParams` 构造器的 guard clause
- [ ] **R1-2** `AesGcmCipher` 的 nonce 使用 `SecureRandom` 生成，不使用计数器或时间戳派生
- [ ] **R1-3** GCM tag 长度为 128 bit（`GCMParameterSpec(128, nonce)`），未被缩短
- [ ] **R1-4** `crypto.json` 的 `integrity` 字段使用 HKDF 派生的子密钥计算，未直接使用 masterKey
- [ ] **R1-5** `CryptoJsonCacheValidator` 在 crypto.json hash 不一致时确实抛出异常并阻止同步继续，不仅仅是打 log
- [ ] **R1-6** 导出加密密码在 `ExportDialogViewModel` 中只存于内存，`onCleared()` 后无法从 DataStore 读取
- [ ] **R1-7** 同步加密密码（masterKey）在 Android Keystore 保护的 DataStore 中存储，不以明文形式写入 `SharedPreferences`

### 隐私与凭据

- [ ] **R1-8** `WebDavClient` 的日志输出不包含密码明文，Authorization header 值在日志中被脱敏（如截断 `Basic ***`）
- [ ] **R1-9** `preferences.json` 导出内容不包含 `webdavUrl`、`webdavUser`、`webdavPassword`、`syncMethod`（见 §5.6 排除项）
- [ ] **R1-10** 远端路径（`books/{bookKey}/`）中不出现书名、作者等任何明文敏感信息

## 阶段 R2 · 真实设备同步流程

> 需要至少 **2 台真实 Android 设备**（推荐 API 26+），接入同一坚果云账号测试。

### 基础连通性

- [ ] **R2-1** 设备 A 输入坚果云账号后，"测试连接"在 5 秒内返回成功状态，卡片显示绿色"已连接"

- [ ] **R2-3** 断网状态下点击"立即同步"，2 秒内出现错误提示，同步状态恢复 IDLE，不永久 loading

### 进度同步

- [ ] **R2-4** 设备 A 阅读《测试书》到第 12 章，关闭阅读页 → 点击"立即同步" → 设备 B 同步后，打开同一书到第 12 章（误差 ≤ 5%）
- [ ] **R2-5** 两设备同时阅读不同章节，其中一台进度落后超过 5%，同步时**弹出冲突弹窗**，不静默覆盖
- [ ] **R2-6** 冲突弹窗中显示的章节信息与实际阅读位置一致（不显示 0 或空）
- [ ] **R2-7** 选择"暂不处理"后，下次启动 App 再同步仍会弹出冲突弹窗（未被静默处理）

### 书签与笔记同步

- [ ] **R2-8** 设备 A 添加 3 条书签，同步后设备 B 同步，设备 B 能看到完整 3 条书签
- [ ] **R2-9** 设备 B 删除设备 A 的其中一条书签，同步后设备 A 该书签消失（tombstone 正确传播）
- [ ] **R2-10** 设备 A 和设备 B 各自添加不同书签，同步后两台设备均能看到双方的书签（不互相覆盖）
- [ ] **R2-11** 笔记同步与书签同步行为一致（重复 R2-8 ~ R2-10 流程验证笔记）

### 配置同步

- [ ] **R2-12** 设备 A 修改字号为 20，设备 B 修改主题为"深色"，同步后两台设备**字号和主题各自保留**（key-level merge 生效）
- [ ] **R2-13** 设备 A 未修改任何配置，同步后设备 A 的配置不被远端覆盖（脏标记未触发）
